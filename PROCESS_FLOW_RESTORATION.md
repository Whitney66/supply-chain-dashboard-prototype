## 🔧 实施方案

由于原始的 Figma 图片资源不可用（`figma:asset` 资源缺失），我们采用了**更优的解决方案**：创建智能SVG流程图组件，而非依赖外部图片。

### 优势对比

| 维度 | 原方案（图片） | 新方案（SVG组件） |
|------|--------------|-----------------|
| 加载速度 | ❌ 慢（需下载图片） | ✅ 快（无需额外请求） |
| 可维护性 | ❌ 难（需重新导出图片） | ✅ 易（直接修改配置） |
| 一致性 | ⚠️ 依赖设计稿 | ✅ 代码控制，统一风格 |
| 响应式 | ⚠️ 可能模糊 | ✅ SVG矢量完美缩放 |
| 文件大小 | ❌ 大（多张图片） | ✅ 小（一个组件） |
| 国际化 | ❌ 难（需重新制图） | ✅ 易（文字可配置） |

## 📦 新增组件

### 1. ProcessFlowSVG.tsx
**功能**: 基础SVG流程图渲染引擎

**特性**:
- 自动计算布局（节点宽度、间距、连接线）
- 智能文字截断（超过8字符自动省略）
- 颜色区分（起点蓝色、终点绿色、中间节点灰色）
- 带箭头的连接线
- 响应式设计（自适应容器宽度）

**参数**:
```typescript
interface ProcessFlowSVGProps {
  steps: string[];      // 流程步骤数组，如 ['供应商', '物流', '仓库']
  className?: string;   // 可选的CSS类名
}
```

### 2. SmartProcessFlow.tsx
**功能**: 智能流程图组件（业务逻辑层）

**特性**:
- 根据 `flowId` 自动匹配对应的业务流程
- 内置11种预定义流程（订货、分货、调拨）
- 美化的渐变背景
- 显示流程说明文字

**参数**:
```typescript
interface SmartProcessFlowProps {
  flowId: string;       // 流程ID，如 'flow_1_1_1'
  alt: string;          // 流程描述文字
  className?: string;   // 可选的CSS类名
}
```

## 🚀 如何扩展

### 添加新的流程图

**步骤1**: 在 `SmartProcessFlow.tsx` 的 `flowMap` 中添加新条目

```typescript
const flowMap: Record<string, string[]> = {
  // ... 现有流程 ...
  
  // 新增流程
  'flow_4_1_1': ['退货申请', '质检', '入库', '退款'],
  'flow_4_2_1': ['换货申请', '发货', '收货', '完成'],
};
```

**步骤2**: 在业务组件中使用

```tsx
<SmartProcessFlow 
  flowId="flow_4_1_1" 
  alt="4.1.1退货流程" 
/>
```

### 自定义流程步骤

如果需要动态流程，可以直接使用底层组件：

```tsx
import { ProcessFlowSVG } from '@/app/components/ProcessFlowSVG';

<ProcessFlowSVG 
  steps={['步骤A', '步骤B', '步骤C']} 
/>
```
