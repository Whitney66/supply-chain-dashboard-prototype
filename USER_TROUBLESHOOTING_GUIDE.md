# 供应链物流数据看板 - 用户故障排查指南

## 🚨 如果页面打不开或显示异常，请按以下步骤操作

---

## 第一步：基础检查 (1分钟)

### ✅ 检查网络连接
- 打开其他网站（如 baidu.com）确认网络正常
- 如果使用公司网络，尝试切换到手机热点

### ✅ 检查浏览器
**推荐使用的浏览器（按推荐顺序）：**
1. ⭐ Google Chrome (120+版本)
2. ⭐ Microsoft Edge (120+版本)
3. ✅ Safari (17+版本)
4. ✅ Firefox (120+版本)

**不推荐的浏览器：**
- ❌ IE 浏览器（任何版本）
- ❌ 360浏览器兼容模式
- ❌ QQ浏览器（可能有兼容性问题）

### ✅ 快速修复三步法
```
1️⃣ 按 Ctrl+F5 (Mac: Cmd+Shift+R) 强制刷新页面
2️⃣ 清除浏览器缓存（见下方详细步骤）
3️⃣ 使用无痕/隐私模式打开（Ctrl+Shift+N 或 Cmd+Shift+N）
```

---

## 第二步：清除浏览器缓存 (2分钟)

### Chrome / Edge 浏览器
1. 按 `Ctrl + Shift + Delete` (Mac: `Cmd + Shift + Delete`)
2. 选择"时间范围"为"过去 1 小时"或"全部"
3. 勾选"缓存的图片和文件"
4. 点击"清除数据"
5. 刷新页面

### Safari 浏览器
1. 菜单栏 → 开发 → 清空缓存
2. 或按 `Cmd + Option + E`
3. 刷新页面

### Firefox 浏览器
1. 按 `Ctrl + Shift + Delete`
2. 选择"缓存"
3. 点击"立即清除"
4. 刷新页面

---

## 第三步：检查错误信息 (3分钟)

### 打开开发者工具
**Windows**: 按 `F12` 或 `Ctrl + Shift + I`  
**Mac**: 按 `Cmd + Option + I`

### 查看 Console (控制台) 标签
**寻找以下错误提示：**

#### ❌ 如果看到红色错误信息

**错误类型 1**: `Failed to fetch` 或 `Network Error`
```
✅ 解决方案：
- 检查防火墙设置
- 联系IT部门将以下域名加入白名单：
  • *.figma.com
  • *.figma.site
  • *.makeproxy-c.figma.site
```

**错误类型 2**: `TypeError: xxx is not defined`
```
✅ 解决方案：
- 清除缓存后强制刷新（Ctrl+F5）
- 如仍有问题，请截图发送给技术支持
```

**错误类型 3**: `ChunkLoadError` 或资源加载失败
```
✅ 解决方案：
- 网络不稳定导致，刷新页面即可
- 检查是否开启了广告拦截插件（建议关闭）
```

### 查看 Network (网络) 标签
1. 刷新页面
2. 查看是否有红色的失败请求
3. 如果有大量 404 或 500 错误，截图发送给技术支持

---

## 第四步：检查Figma权限 (仅首次访问用户)

### 症状
- 提示"无权限访问"
- 页面显示登录界面
- 看到"404 Not Found"

### 解决方案

#### 方案A：检查链接权限设置（分享者操作）
1. 打开 Figma 文件
2. 点击右上角 **"Share"** 按钮
3. 确保设置为：**"Anyone with the link can view"** （任何人可查看）
4. 重新分享链接给用户

#### 方案B：登录Figma账号（访问者操作）
1. 访问 https://figma.com
2. 使用邮箱登录或注册（免费）
3. 登录后重新打开链接

## 常见问题 FAQ

### Q1: 页面显示空白，没有任何内容
**A**: 
1. 检查是否是网络加载过慢，等待30秒
2. 按 F12 打开控制台查看是否有错误
3. 清除缓存后重试
4. 更换浏览器测试

### Q2: 某些Tab（指标总览/指标明细/异常分析）无法切换
**A**:
1. 这通常是组件加载错误，查看控制台错误信息
2. 刷新页面
3. 如果只有某一个Tab有问题，截图发送给技术支持

### Q3: 图表不显示，只有空白框
**A**:
1. 可能是浏览器不支持 SVG 渲染
2. 更新浏览器到最新版本
3. 关闭硬件加速：
   - Chrome: 设置 → 高级 → 系统 → 关闭"使用硬件加速"

### Q4: 筛选器（日期/门店/仓库）无法展开
**A**:
1. 检查是否有JavaScript错误（按F12查看Console）
2. 禁用浏览器扩展（广告拦截器可能干扰）
3. 使用无痕模式测试

### Q5: 数据显示为"-"或"暂无数据"
**A**:
1. 这是正常的，说明该筛选条件下没有数据
2. 尝试调整筛选条件（日期范围、门店选择）
3. 确认是否有数据权限

### Q6: 页面提示"组件加载失败"
**A**:
1. 点击页面上的"刷新页面"按钮
2. 如果反复出现，可能是某个组件有bug
3. 截图错误信息发送给技术支持

## 预防措施

### ✅ 日常使用建议
1. **使用推荐浏览器** - Chrome 或 Edge 最新版
2. **定期清理缓存** - 每周清理一次浏览器缓存
3. **保持浏览器更新** - 开启自动更新功能
4. **稳定的网络环境** - 避免频繁切换网络
5. **保存常用链接** - 添加到浏览器书签

### ⚠️ 避免以下操作
1. ❌ 不要使用过时的IE浏览器
2. ❌ 不要在浏览器开启过多标签页（建议<20个）
3. ❌ 不要使用破解版或修改版浏览器
4. ❌ 不要在网络不稳定时进行重要操作

## 技术原理说明（可选阅读）

### 为什么有人能打开，有人打不开？

**主要原因有3个：**

1. **权限问题** (50%)
   - Figma文件未设置公开访问权限
   - 需要登录Figma账号才能查看

2. **网络问题** (30%)
   - 公司防火墙屏蔽了Figma域名
   - DNS解析失败
   - 网络不稳定导致资源加载超时

3. **浏览器兼容性** (20%)
   - 使用了不支持的旧版浏览器
   - 浏览器缓存损坏
   - 浏览器扩展冲突

### 应用架构说明
```
用户浏览器
    ↓
Figma Make 托管服务 (*.figma.site)
    ↓
React 单页应用
    ├── 指标总览页面
    ├── 指标明细页面
    └── 异常分析页面
```

所有数据和组件都在首次加载时下载到浏览器，
如果首次加载成功，后续切换Tab应该都能正常工作。
